---
title: 'Process a de-freeze request'
description: 'Bank officer approves or records a decision on a verified de-freeze request.'
---

<Note>
  This endpoint requires authentication. Pass the JWT token returned by [Log in](/api/auth/login) in the `token` request header. The caller's `usertype` must be `"Bank"`.
</Note>

## Request

**Method:** `POST`  
**Path:** `/formupload/defreezebybank`

### Header parameters

<ParamField header="token" type="string" required>
  A valid JWT session token obtained from the [login endpoint](/api/auth/login). The `usertype` encoded in the token must be `"Bank"`.
</ParamField>

### Body parameters

<ParamField body="applicationNo" type="number" required>
  The unique identifier of the complaint whose de-freeze request you are processing. Obtain pending cases from the [list pending de-freeze cases](/api/defreeze/status#get-formuploaddownloadformsnotverifyfordefreeze) endpoint.
</ParamField>

<ParamField body="verifybypolice" type="string" required>
  The bank officer's de-freeze decision. This field is stored as `defreezestatus` on the complaint document.

  Pass the decision string as required by your workflow (e.g. `"approved"` or `"rejected"`). The value is stored verbatim.
</ParamField>

## Pre-conditions

The endpoint only updates complaints that already exist in the database — it does not enforce a status filter itself. Use the [list pending de-freeze cases](/api/defreeze/status#get-formuploaddownloadformsnotverifyfordefreeze) endpoint to retrieve cases that are ready for bank action. Those cases meet all of the following criteria:

- `verifybypolice: "yes"` — police have verified the complaint
- `verifybypoliceemail` is set
- `bankbool: true` — the complaint involves a bank transaction
- `verifybycourt: "yes"` — court has verified the complaint
- `verifybycourtemail` is set

<Warning>
  If the token's `usertype` is not `"Bank"`, or if no complaint matches the provided `applicationNo`, the server returns `false` and no changes are made.
</Warning>

## How it works

On a successful call the server sets two fields on the matching complaint document:

| Field | Value |
|---|---|
| `defreezestatus` | The value from `req.body.verifybypolice` |
| `defreezestatusbyemail` | The email address of the authenticated bank officer, extracted from the token |

## Response

All responses return HTTP `200` with a JSON body.

<ResponseField name="(root)" type="boolean">
  `true` when the complaint was found and the de-freeze decision was saved. `false` when the complaint does not exist, the caller is not a bank officer, or a server error occurs.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://your-api-host/formupload/defreezebybank \
  --header 'Content-Type: application/json' \
  --header 'token: <bank-jwt-token>' \
  --data '{
    "applicationNo": 4821,
    "verifybypolice": "approved"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-api-host/formupload/defreezebybank', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    token: '<bank-jwt-token>',
  },
  body: JSON.stringify({
    applicationNo: 4821,
    verifybypolice: 'approved',
  }),
});

const success = await response.json();
console.log(success); // true
```

</CodeGroup>

### Success response

```json
true
```

### Failure — complaint not found, unauthorised role, or server error

```json
false
```
