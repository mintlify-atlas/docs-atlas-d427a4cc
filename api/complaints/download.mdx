---
title: 'List and retrieve complaints'
description: 'Retrieve complaint records and proof images for the authenticated user or role.'
---

<Note>
  All endpoints on this page require authentication. Pass the JWT token returned by [Log in](/api/auth/login) in the `token` request header.
</Note>

## Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/formupload/downloadforms` | List complaints for the authenticated user or role. |
| `GET` | `/formupload/downloadformsnotverify` | List complaints pending verification (police, bank, and court use). |
| `GET` | `/formupload/image` | Retrieve the proof image for the authenticated user's most recent complaint. |

---

## GET /formupload/downloadforms

Returns complaint records scoped to the authenticated user's role:

- **Victim** — all complaints filed by that victim's email address.
- **Police** — complaints the officer has already verified (where `verifybypoliceemail` matches the officer's email).
- **Bank** — complaints the bank officer has already verified (police-verified, bank-involved, and `verifybybankemail` matches).
- **Court** — complaints the court officer has already verified (police-verified and `verifybycourtemail` matches).

Returns `false` if the token belongs to an unrecognised role or if an error occurs.

### Header parameters

<ParamField header="token" type="string" required>
  A valid JWT session token obtained from the [login endpoint](/api/auth/login).
</ParamField>

### Response

An array of `incidentform` documents. Each document has the following fields:

<ResponseField name="applicationNo" type="number">
  The unique 4-digit identifier for the complaint.
</ResponseField>

<ResponseField name="email" type="string">
  The email address of the victim who filed the complaint.
</ResponseField>

<ResponseField name="type" type="string">
  The complaint classification assigned by the ML model.
</ResponseField>

<ResponseField name="category" type="string">
  The top-level crime category (always `"Online Financial Fraud"`).
</ResponseField>

<ResponseField name="subcategory" type="string">
  The specific fraud subcategory selected by the victim.
</ResponseField>

<ResponseField name="delay" type="boolean">
  `true` if the complaint was filed with a delay after the incident.
</ResponseField>

<ResponseField name="bankbool" type="boolean">
  `true` if a bank account or transaction is involved.
</ResponseField>

<ResponseField name="bank" type="object">
  Bank transaction details. Present when `bankbool` is `true`.

  <Expandable title="properties">
    <ResponseField name="bank.amount" type="number">
      The fraudulent amount in Indian Rupees (INR).
    </ResponseField>
    <ResponseField name="bank.name" type="string">
      The name of the bank or financial institution involved.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="date" type="string">
  The date the incident occurred.
</ResponseField>

<ResponseField name="incidentoccur" type="string">
  The platform or medium where the incident took place.
</ResponseField>

<ResponseField name="whereid" type="string">
  The suspect's identifier on the incident platform.
</ResponseField>

<ResponseField name="description" type="string">
  The victim's narrative of the incident.
</ResponseField>

<ResponseField name="proof" type="string">
  Base64-encoded documentary evidence uploaded by the victim.
</ResponseField>

<ResponseField name="legitment" type="string">
  Legitimacy assessment produced by the ML model.
</ResponseField>

<ResponseField name="verifybypolice" type="string">
  Police verification decision. `"yes"` when verified, absent if not yet reviewed.
</ResponseField>

<ResponseField name="verifybypoliceemail" type="string">
  Email address of the police officer who verified the complaint.
</ResponseField>

<ResponseField name="verifybybank" type="string">
  Bank verification decision. `"yes"` when verified, absent if not yet reviewed.
</ResponseField>

<ResponseField name="verifybybankemail" type="string">
  Email address of the bank officer who verified the complaint.
</ResponseField>

<ResponseField name="verifybycourt" type="string">
  Court verification decision. `"yes"` when verified, absent if not yet reviewed.
</ResponseField>

<ResponseField name="verifybycourtemail" type="string">
  Email address of the court officer who verified the complaint.
</ResponseField>

<ResponseField name="defreezestatus" type="string">
  Current status of any de-freeze request linked to this complaint.
</ResponseField>

## GET /formupload/downloadformsnotverify

Returns complaints that are pending verification, filtered by the caller's role:

- **Police** — complaints that have not yet been reviewed by any police officer (no `verifybypolice` or `verifybypoliceemail` fields present).
- **Bank** — police-verified complaints (`verifybypolice: "yes"`) involving a bank (`bankbool: true`) that have not yet been reviewed by a bank officer.
- **Court** — police-verified complaints that have not yet been reviewed by a court officer.

Returns `false` for any other role.

<Warning>
  This endpoint is intended for police, bank, and court officers. Victim accounts receive `false`.
</Warning>

### Header parameters

<ParamField header="token" type="string" required>
  A valid JWT session token obtained from the [login endpoint](/api/auth/login).
</ParamField>

### Response

Same structure as the `/downloadforms` response array above.

## GET /formupload/image

Returns the first complaint record (including `proof`) associated with the authenticated user's email address. Intended for displaying the proof image stored in a complaint.

<Note>
  This endpoint returns the first matching complaint document, not a standalone image file. The `proof` field within the response contains the Base64-encoded image.
</Note>

### Header parameters

<ParamField header="token" type="string" required>
  A valid JWT session token obtained from the [login endpoint](/api/auth/login).
</ParamField>

### Response

A single `incidentform` document (same fields as listed above), or `false` if no complaint is found for the authenticated email or the token is invalid.

## Examples

<CodeGroup>

```bash cURL — list complaints
curl --request GET \
  --url https://your-api-host/formupload/downloadforms \
  --header 'token: <your-jwt-token>'
```

```javascript JavaScript — list complaints
const response = await fetch('https://your-api-host/formupload/downloadforms', {
  method: 'GET',
  headers: {
    token: '<your-jwt-token>',
  },
});

const complaints = await response.json();
console.log(complaints); // array of incidentform documents
```

</CodeGroup>

<CodeGroup>

```bash cURL — list unverified complaints
curl --request GET \
  --url https://your-api-host/formupload/downloadformsnotverify \
  --header 'token: <police-jwt-token>'
```

```javascript JavaScript — list unverified complaints
const response = await fetch(
  'https://your-api-host/formupload/downloadformsnotverify',
  {
    method: 'GET',
    headers: {
      token: '<police-jwt-token>',
    },
  }
);

const pending = await response.json();
console.log(pending); // array of complaints pending verification
```

</CodeGroup>

<CodeGroup>

```bash cURL — get proof image
curl --request GET \
  --url https://your-api-host/formupload/image \
  --header 'token: <your-jwt-token>'
```

```javascript JavaScript — get proof image
const response = await fetch('https://your-api-host/formupload/image', {
  method: 'GET',
  headers: {
    token: '<your-jwt-token>',
  },
});

const record = await response.json();
if (record && record.proof) {
  // record.proof is a Base64-encoded string
  const imageSrc = record.proof;
}
```

</CodeGroup>
