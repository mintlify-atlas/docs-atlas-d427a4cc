---
title: 'Submit a complaint'
description: 'Submit a new cybercrime complaint to the CyberCrew 1930 Helpline Platform.'
---

<Note>
  This endpoint requires authentication. Pass the JWT token returned by [Log in](/api/auth/login) in the `token` request header.
</Note>

## Request

**Method:** `POST`  
**Path:** `/formupload/upload`

### Header parameters

<ParamField header="token" type="string" required>
  A valid JWT session token obtained from the [login endpoint](/api/auth/login).
</ParamField>

### Body parameters

<ParamField body="category" type="string" required>
  The top-level crime category. Always `"Online Financial Fraud"`.
</ParamField>

<ParamField body="subcategory" type="string" required>
  The specific type of financial fraud. One of:

  - `"Online Banking, Credit/Debit Card & Sim Swap Fraud"`
  - `"Business Email Compromise/Email Takeover"`
  - `"E-Wallet Related Fraud"`
  - `"Fraud Call/Vishing"`
  - `"Demat/Depository Fraud"`
  - `"Online Job Fraud"`
  - `"Online Matrimonial Fraud"`
  - `"UPI-Related Frauds"`
</ParamField>

<ParamField body="delay" type="string" required>
  Whether the complaint is being filed with a delay after the incident. `"yes"` or `"no"`.
</ParamField>

<ParamField body="bankbool" type="string" required>
  Whether a bank account or transaction is involved. `"yes"` or `"no"`. When `"yes"`, the `bank` field must be provided.
</ParamField>

<ParamField body="bank" type="object">
  Bank transaction details. Required when `bankbool` is `"yes"`.

  <Expandable title="properties">
    <ParamField body="bank.amount" type="number" required>
      The fraudulent amount transferred, in Indian Rupees (INR).
    </ParamField>

    <ParamField body="bank.name" type="string" required>
      The name of the bank or financial institution involved.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="date" type="string" required>
  The date the incident occurred. ISO 8601 format recommended (e.g., `"2024-03-15"`).
</ParamField>

<ParamField body="incidentoccur" type="string" required>
  The platform or medium on which the incident took place (e.g., `"WhatsApp"`, `"Email"`, `"Phone Call"`).
</ParamField>

<ParamField body="whereid" type="string" required>
  The suspect's identifier on the platform where the incident occurred (e.g., a phone number, username, or email address).
</ParamField>

<ParamField body="description" type="string" required>
  A detailed narrative of the incident as described by the victim.
</ParamField>

<ParamField body="proof" type="string" required>
  Documentary evidence of the fraud encoded as a Base64 string. Accepted formats include images (JPEG, PNG) and PDFs.
</ParamField>

<ParamField body="emailtakenby" type="string">
  The email address used by the attacker to impersonate a legitimate sender. Only applicable when `subcategory` is `"Business Email Compromise/Email Takeover"`.
</ParamField>

<ParamField body="header" type="string">
  The raw email header from the fraudulent message, used for forensic analysis. Only applicable when `subcategory` is `"Business Email Compromise/Email Takeover"`.
</ParamField>

## How it works

When a complaint is submitted, the server:

1. Extracts the authenticated user's email from the JWT token and attaches it to the record.
2. Generates a unique 4-digit `applicationNo` for tracking the complaint.
3. Forwards the complaint data to the internal ML service (`/predict`) to assess fraud legitimacy and classify the complaint type.
4. Stores the resulting document — including ML predictions — in the database.

<Tip>
  The `applicationNo` returned in the response is the primary identifier used for all subsequent operations on this complaint, including verification and de-freeze requests.
</Tip>

## Response

All responses return HTTP `200` with a JSON body.

On success the server returns the submitted request body enriched with the fields assigned during processing.

<ResponseField name="applicationNo" type="number">
  The unique 4-digit identifier generated for this complaint.
</ResponseField>

<ResponseField name="email" type="string">
  The email address of the authenticated user who filed the complaint (extracted from the token).
</ResponseField>

<ResponseField name="type" type="string">
  The complaint type assigned by the ML classification service.
</ResponseField>

<ResponseField name="legitment" type="string">
  A legitimacy assessment produced by the ML model indicating whether the complaint appears genuine.
</ResponseField>

Returns `false` (boolean) when the submission fails — for example, due to a database error or an invalid token.

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://your-api-host/formupload/upload \
  --header 'Content-Type: application/json' \
  --header 'token: <your-jwt-token>' \
  --data '{
    "category": "Online Financial Fraud",
    "subcategory": "UPI-Related Frauds",
    "delay": "no",
    "bankbool": "yes",
    "bank": {
      "amount": 15000,
      "name": "State Bank of India"
    },
    "date": "2024-03-14",
    "incidentoccur": "WhatsApp",
    "whereid": "+91 98765 43210",
    "description": "Received a call from an unknown number claiming to be from SBI. Was asked to share OTP to unblock account.",
    "proof": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..."
  }'
```

```javascript JavaScript
const response = await fetch('https://your-api-host/formupload/upload', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    token: '<your-jwt-token>',
  },
  body: JSON.stringify({
    category: 'Online Financial Fraud',
    subcategory: 'UPI-Related Frauds',
    delay: 'no',
    bankbool: 'yes',
    bank: {
      amount: 15000,
      name: 'State Bank of India',
    },
    date: '2024-03-14',
    incidentoccur: 'WhatsApp',
    whereid: '+91 98765 43210',
    description:
      'Received a call from an unknown number claiming to be from SBI. Was asked to share OTP to unblock account.',
    proof: 'data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...',
  }),
});

const data = await response.json();
console.log(data.applicationNo); // e.g. 4821
```

</CodeGroup>
